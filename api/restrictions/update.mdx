---
title: Update Restriction Rule
description: Modify existing restriction rules (amount limits, cutoffs, active status)
---

## Overview

Update an existing restriction rule. Only specified fields are modified (partial update).

## Endpoint

```http
PATCH /api/v1/restrictions/:id
```

**Authorization:** ADMIN only

## Path Parameters

<ParamField path="id" type="string" required>
  UUID of the restriction rule to update
</ParamField>

## Request Body

<Warning>
  **Cannot modify:** `scope`, `entityId` (bancaId/ventanaId/userId), `loteriaId`, or `number` after creation.
  
  To change these fields, delete and recreate the rule instead.
</Warning>

### Updatable Fields

<ParamField body="restrictionType" type="string">
  Rule type label
</ParamField>

<ParamField body="isActive" type="boolean">
  Enable or disable the rule without deleting it
</ParamField>

<ParamField body="isAutoDate" type="boolean">
  Enable/disable automatic date synchronization
</ParamField>

<ParamField body="number" type="string">
  **Legacy single-number format only** (batch updates not supported)
  
  To update multiple numbers, delete and recreate.
</ParamField>

<ParamField body="maxAmount" type="number">
  Maximum bet amount per jugada (must be positive)
</ParamField>

<ParamField body="maxTotal" type="number">
  Maximum total amount per ticket (must be positive)
</ParamField>

<ParamField body="baseAmount" type="number">
  Base amount for dynamic limits (≥ 0)
</ParamField>

<ParamField body="salesPercentage" type="number">
  Sales percentage for dynamic limits (0-100)
</ParamField>

<ParamField body="appliesToVendedor" type="boolean">
  Whether percentage applies per-vendedor
</ParamField>

<ParamField body="salesCutoffMinutes" type="number">
  Minutes before draw to stop sales (0-30)
</ParamField>

<ParamField body="appliesToDate" type="string">
  ISO 8601 date filter (or null to remove filter)
</ParamField>

<ParamField body="appliesToHour" type="number">
  Hour filter 0-23 (or null to remove filter)
</ParamField>

<ParamField body="multiplierId" type="string">
  Multiplier UUID
</ParamField>

<ParamField body="message" type="string">
  Custom message (1-255 characters, or null to remove)
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Updated restriction rule (see List endpoint for full schema)
</ResponseField>

## Validation Rules

<AccordionGroup>
  <Accordion title="Cutoff rules cannot mix with other types">
    Cannot update a cutoff rule to have amount limits, or vice versa.
  </Accordion>
  
  <Accordion title="Auto-date validation">
    If setting `isAutoDate = true`:
    - `number` field must be omitted or null
    - Rule must have `maxAmount` or `maxTotal`
  </Accordion>
  
  <Accordion title="Dynamic limits validation">
    `salesPercentage` requires:
    - Value between 0-100
    - Used with amount-based rules only
    
    `appliesToVendedor` only valid when `salesPercentage` is present.
  </Accordion>
</AccordionGroup>
